===================================================================
INFRASTRUCTURE DECISION — TRAEFIK REPLACES NGINX
===================================================================

Use Traefik v3 as the reverse proxy and TLS termination layer.
Traefik replaces Nginx entirely. No nginx.conf file needed.

Why Traefik:
- Automatic Let's Encrypt certificate provisioning and renewal
- Docker-native: reads labels from docker-compose.yml
- No manual cert copy steps needed
- Hot-reload: no container restart needed for config changes
- Built-in dashboard for monitoring routes
- Automatic HTTP → HTTPS redirect via entrypoints

Traefik handles:
- Port 80  → redirect to 443
- Port 443 → TLS termination via Let's Encrypt
- Route /api/* → backend:8000
- Route /      → frontend:3000
- Route /api/track/* → backend:8000 (no cache headers)

Traefik does NOT handle mail ports — those are direct:
- Port 25  → backend container (SMTP inbound)
- Port 587 → backend container (SMTP submission)
- Port 993 → backend container (IMAP SSL)

async def send_direct(from_addr, to_list, subject,
                      body_text, body_html, attachments, headers):
  1. DKIM sign message with sender domain private key
  2. For each recipient:
     a. Check unsubscribe_list — skip if found
     b. dnspython: query MX records for recipient domain
     c. Sort MX by priority, try each until success
     d. aiosmtplib.send with STARTTLS
     e. On all MX fail: raise SMTPException for Celery retry
  3. Return {success, message_id, failed_recipients}

infra/dkim/generate_keys.sh:
  #!/bin/bash
  DOMAIN=$1
  if [ -z "$DOMAIN" ]; then echo "Usage: $0 <domain>"; exit 1; fi
  mkdir -p infra/dkim
  openssl genrsa -out infra/dkim/${DOMAIN}.private 2048
  openssl rsa -in infra/dkim/${DOMAIN}.private -pubout -out infra/dkim/${DOMAIN}.public
  PUBLIC=$(grep -v '^-' infra/dkim/${DOMAIN}.public | tr -d '\n')
  echo ""
  echo "=== DKIM DNS Record for ${DOMAIN} ==="
  echo "Type: TXT"
  echo "Name: mail._domainkey.${DOMAIN}"
  echo "Value: v=DKIM1; k=rsa; p=${PUBLIC}"
  echo ""

# MailOS — Self-Hosted Email Platform

## Prerequisites
- Contabo VPS Ubuntu 24.04 (port 25 open by default)
- Domain with DNS pointed to server IP
- Docker + Docker Compose installed

## Quick Start

1. Clone repo to server
2. cp .env.example .env — fill all values
3. Generate secrets:
   JWT_SECRET_KEY: openssl rand -hex 32
   ENCRYPTION_SECRET_KEY: openssl rand -hex 32
4. Set PTR record in Contabo panel → mail.yourdomain.com
5. Add DNS records at Cloudflare:
   A     mail → YOUR_IP
   MX    @    → mail.yourdomain.com (priority 10)
   TXT   @    → v=spf1 ip4:YOUR_IP mx ~all
   TXT   _dmarc → v=DMARC1; p=quarantine; rua=mailto:dmarc@yourdomain.com
6. Generate DKIM keys: make dkim DOMAIN=yourdomain.com
   Add the TXT record output to Cloudflare DNS
7. make build
8. Traefik automatically obtains Let's Encrypt certificate
   (Requires port 80 open for HTTP challenge)
9. make migrate
10. make seed
11. Login at https://yourdomain.com/login

## How Traefik TLS Works
Traefik automatically requests and renews Let's Encrypt certificates.
Certificates stored in traefik/acme/ volume.
No manual certbot commands needed for HTTPS.
SMTP/IMAP certs still require separate provisioning (Certbot on host).

## Mail Port TLS (SMTP/IMAP)
These ports bypass Traefik — handled directly by backend:
  Port 25:  SMTP inbound (Contabo open)
  Port 587: SMTP submission (STARTTLS)
  Port 993: IMAP (TLS from connect)
For these, run on the host:
  certbot certonly --standalone -d mail.yourdomain.com
  cp /etc/letsencrypt/live/mail.yourdomain.com/fullchain.pem infra/mail-certs/
  cp /etc/letsencrypt/live/mail.yourdomain.com/privkey.pem infra/mail-certs/
  Mount infra/mail-certs as /etc/ssl/mail in backend container.

## Traefik Dashboard
Access at: https://traefik.yourdomain.com
Protected by basic auth (set TRAEFIK_DASHBOARD_PASSWORD_HASH in .env).

## Roles
  /login → super_admin   → /super-admin
  /login → domain_admin  → /domain-admin
  /login → user          → /mail/inbox

## Upgrade
  git pull && make build && make migrate

SECTION 18 — SMTP/IMAP CERT HANDLING IN DOCKER
===================================================================

Since Traefik does not proxy raw TCP (mail ports), SMTP and IMAP
need their own TLS certificates.

Add to infra/ directory:
  infra/mail-certs/
    fullchain.pem   (from certbot or Let's Encrypt)
    privkey.pem

Mount in docker-compose backend service:
  - ./infra/mail-certs:/etc/ssl/mail:ro

Add to Makefile:
mail-cert:
  certbot certonly --standalone -d mail.$(DOMAIN) \
    --email $(ACME_EMAIL) --agree-tos --non-interactive
  cp /etc/letsencrypt/live/mail.$(DOMAIN)/fullchain.pem infra/mail-certs/
  cp /etc/letsencrypt/live/mail.$(DOMAIN)/privkey.pem infra/mail-certs/
  chmod 644 infra/mail-certs/*.pem

mail-cert-renew:
  certbot renew --quiet
  cp /etc/letsencrypt/live/mail.$(DOMAIN)/fullchain.pem infra/mail-certs/
  cp /etc/letsencrypt/live/mail.$(DOMAIN)/privkey.pem infra/mail-certs/
  docker compose restart backend

Add cron for auto-renewal:
  0 3 * * * cd /home/deploy/email-platform && make mail-cert-renew
